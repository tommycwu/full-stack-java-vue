Please make sure you have read the entire Case Study before attemping the tasks in this Use Case.

1. Set up Okta Org2Org using SAML 2.0 as the sign-on method.


<Note: Be sure to search for and select the Okta Org2Org application in the Okta Integration Network. Do NOT change the application name from the default.

2. Set up Okta Lifecycle Management for the Okta Org2Org application defined in Step 1. Enable the following to App provisioning options:

Create Users

Update User Attributes

Deactivate Users

Note: For the Security token, use the API Key provided on the Org Info tab.

3. Assign the Interns group to the Okta Org2Org app.

4. Verify that Mike's identity is pushed to org2 successfully.

Important: If Mike's identity is NOT pushed to org2 successfully. Step 5 and the first section of Use Case 4 will fail.

5. Log in to Okta as Mike and verify he can access org2 by clicking the Okta Org2Org icon in his Okta Dashboard.
